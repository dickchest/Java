Уровень 1
1. Что такое immutable?
"Immutable" в Java означает, что объект не может быть изменен после создания. То есть, если объект является неизменяемым, любые попытки изменения его состояния приведут к созданию нового объекта.

2. Для чего нужны immutable классы?
- Безопасность: если объект не может быть изменен после создания, то можно предоставить доступ к нему другим частям программы без риска изменения его состояния, что может быть важно для безопасности.
- Потокобезопасность: immutable объекты могут безопасно использоваться в многопоточных приложениях, потому что их состояние не может быть изменено одним потоком, в то время как другой поток работает с тем же объектом.
- Кэширование: объекты с постоянным состоянием могут быть кэшированы и повторно использованы, что может улучшить производительность программы.
- Простота кода: если объект не может быть изменен, то это упрощает код и уменьшает возможность ошибок, связанных с изменением его состояния.

3. Что дает ключевое слово final?
- для переменных - определение констант
- для классов - класс не может быть расширен другими классами
- для метордов - поведение не может быть изменено в производных классах
- для аргумента - значение аргумента для метода не может быть изменено внутри метода
